## 9. 拟物复刻：单反相机（Camera Replica）

> 「拟物复刻」题型：SVG + CSS 纯手绘、不设行数限制，核心考察“画得有多像”。命门是反作弊：禁止任何贴图/emoji 代替绘制，违反即判 0 分。

```text
你是一个参加“拟物复刻”视觉压测的模型。请按以下硬约束输出作品：

[硬约束]
1. 仅允许使用: HTML + CSS + SVG + JavaScript
2. 仅输出一个文件: `index.html`
3. 不设代码行数限制，把目标物体画到极致，追求以假乱真的拟物质感
4. 禁止外部依赖（禁止 npm 包、禁止 CDN、禁止远程资源、禁止网络字体）
5. 反作弊（本题型核心，违反即判 0 分）：
   - 禁止任何位图：不得使用 <img>、background-image:url()、base64 / data: 图片、<canvas> 贴图
   - 禁止用 emoji 或 Unicode 符号（如 📷📸🎥）代替绘制
   - 禁止引用任何现成 SVG 图标 / 素材 / 图标字体
   - 目标物必须完全由 HTML 元素 + CSS + 内联 SVG 形状（rect/circle/ellipse/path/gradient/filter）纯手工绘制
6. 主体画面必须使用内联 SVG / CSS 绘制，允许内联 <style> 和少量 <script> 增强细节与动效
7. 画面以目标物体为绝对主体、居中呈现、背景干净，移动端宽度 390px 下主体仍清晰、比例不变形
8. 页面必须可直接在浏览器打开运行，不需要构建步骤

[输出格式]
1. 先输出一段 3-5 行“设计说明”（中文）
2. 然后输出完整 index.html 代码块
3. 不要输出额外解释，不要拆分成多个文件

[评分重点（核心是“画得有多像”）]
- 轮廓与比例：整体是否一眼就能认出，比例是否真实协调
- 透视一致性：视角是否自洽、立体结构是否成立、不穿帮
- 材质与光影：金属 / 玻璃 / 塑料等不同质感是否区分，光源是否统一、有高光与投影
- 细节完整度：必做部件是否齐全、细节密度与精致度
- 结构质量：SVG / DOM 分层清晰、渐变与滤镜命名合理、响应式良好
- 约束遵守：是否纯手绘无贴图作弊、单文件、可直接运行

[主题] 拟物复刻：单反相机（3/4 视角）
目标：画出一台 3/4 侧前方视角的单反相机（机身 + 可拆镜头），让人一眼认出并惊叹其拟物质感。本题考察模型在零素材条件下，对真实物体结构、透视、材质与光影的理解与还原能力。

必做功能：
1. 3/4 视角成立：机身为有厚度的矩形体块、带可见侧面透视；镜头正圆在该视角下呈正确的椭圆；机身与镜头衔接自然、不穿帮
2. 镜头分层：多层镜筒（对焦环 / 变焦环纹理）+ 前组镜片，镜片要有同心圆结构与至少一处高光反射
3. 机身部件齐全：军舰头（顶部取景器凸起）、模式转盘（带分段刻度）、快门按钮、热靴、手柄握把（带防滑纹理）
4. 材质区分：在同一光源方向下，明显区分金属（镜环）、玻璃（镜片反光）、磨砂塑料（机身）三种质感
5. 自创（非真实商标）的品牌 / 型号字样，排版精致，且不得使用任何真实相机厂商 logo

加分项：
- 红圈 / 装饰环、对焦距离刻度、螺纹等高密度细节
- 真实的接触投影与环境光遮蔽（相机在台面的落影、部件缝隙暗部）
- 轻微动效：镜片反光流动 / 模式转盘可点击转动（不破坏静态首帧观感）
```

硬性约束（违反任意一条即不合格）：

1. 零外部依赖：禁止引入任何第三方库（Three.js、regl 等一律不允许）、禁止任何图片/视频/3D 模型/预烘焙贴图、禁止外部网络请求（字体除外，且必须有系统字体回退）。所有视觉内容必须由代码实时生成。

2. 真实测地线积分：光线弯曲必须来自片元着色器中对光子测地线的数值积分（施瓦西度规，几何单位制 r_s = 1），不得使用屏幕空间扭曲、法线贴图折射、或任何“看起来像透镜”的近似。积分方案自选（建议守恒角动量形式 acc = -1.5 * h^2 * r / r^5 配合辛欧拉或 RK4），但必须写明所用方案与步长策略的注释。

3. 视界与光子球：r < 1 的光线落入视界呈纯黑阴影；r ≈ 1.5 处应能观察到光子球附近的光线缠绕（阴影边缘的细亮环）。

   - 温度随半径变化的黑体辐射颜色（T ∝ r^(-3/4) 或合理近似）；
   - 差速旋转的可见条纹结构（内快外慢）。

5. 多重像：由于强透镜效应，必须能看到吸积盘绕过黑洞上方与下方形成的次级像（即“光环抱住黑洞”的标志性形态），这应当从积分中自然产生而非手绘。

6. HDR 后期管线：场景渲染到浮点（或可用时的半浮点）帧缓冲，随后至少包含高亮提取、高斯模糊 bloom、色调映射（ACES 或同级）三个独立 pass。设备不支持浮点缓冲时须自动降级而非黑屏。

7. 交互：鼠标拖拽旋转视角（建议带惯性）、滚轮缩放；移动端支持单指拖拽与双指捏合。缩放范围需限制在合理区间，不允许穿入视界。

8. 实时遥测：界面上须有一个数据面板，实时显示当前相机距离处的引力时间膨胀因子 sqrt(1 - r_s/r) 与当地圆轨道速度 beta = sqrt(r_s/2r)，数值随缩放实时更新。这两个数字会被人工验算。

9. 性能与韧性：提供至少两档画质；帧率持续过低时自动降档并提示用户；WebGL 不可用、着色器编译失败、context lost 三种情况都必须有友好的错误界面而非白屏。
