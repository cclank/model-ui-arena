# 模型前端能力对比 - 完整提示词

默认约束：`HTML + CSS + JavaScript`、单文件 `index.html`、总行数 `<= 180`。

## 1. 钟表（Clock）

```text
你是一个参加前端压测的模型。请按以下硬约束输出代码：

[硬约束]
1. 仅允许使用: HTML + CSS + JavaScript
2. 仅输出一个文件: `index.html`
3. 总行数 <= 180
4. 禁止外部依赖（禁止 npm 包、禁止 CDN、禁止远程图片）
5. 允许使用内联 `<style>` 和 `<script>`
6. 页面必须响应式，移动端宽度 390px 下可用
7. 页面必须可直接在浏览器打开运行，不需要构建步骤

[输出格式]
1. 先输出一段 3-5 行“设计说明”（中文）
2. 然后输出完整 `index.html` 代码块
3. 不要输出额外解释，不要拆分成多个文件

[评分重点]
- 创意与视觉完成度
- 交互与状态管理
- 信息层级与可读性
- 代码简洁度与稳定性
- 在约束内完成目标的能力

[主题] 钟表
目标：构建一个“高辨识度”的时钟页面。

必做功能：
1. 显示当前本地时间（时/分/秒）
2. 秒级刷新，不能卡顿
3. 同时展示“模拟元素”与“数字时间”
4. 至少一个动效细节（如秒针、背景律动、刻度呼吸）

加分项：
- 时间段变化（白天/夜晚风格）
- 精致排版与层次
- 小而稳的性能优化
```

1. 至少 4 个输入项（滑块/选择器均可）

- 情境化文案（通勤、周末、自驾）
